# Rencana Kopi Arsip dan Riwayat Pelanggan

## Tujuan
Meniru data tabel arsip dan riwayat pelanggan PT Karya Teknik Mulia ke pelanggan PT Bank Tabungan Negara (Persero) dalam sistem FO KIMA.

## Langkah-langkah

### 1. Identifikasi ID Pelanggan
- Gunakan API Supabase untuk mencari pelanggan dengan nama "PT Bank Tabungan Negara (Persero)" dan ambil ID-nya.
- Gunakan API Supabase untuk mencari pelanggan dengan nama "PT Karya Teknik Mulia" dan ambil ID-nya.

### 2. Ambil Data Arsip dari PT Karya Teknik Mulia
- Pada halaman detail pelanggan PT Karya Teknik Mulia, akses tab "Invoice".
- Filter tampilan untuk menampilkan hanya riwayat arsip (scheduleStatus = "history").
- Catat semua field yang relevan untuk setiap arsip:
  - contractNumber
  - invoiceNumber
  - periodStartDate
  - periodEndDate
  - dueDate
  - paidAt
  - amount
  - packageType
  - ratio
  - coreTotal
  - remarks
  - File yang diunggah (jika ada) - hanya catat referensi, bukan file fisik

### 3. Buat Arsip Baru untuk PT Bank Tabungan Negara (Persero)
- Untuk setiap arsip dari PT Karya Teknik Mulia yang ditemukan:
  - Siapkan payload untuk membuat arsip baru dengan:
    - customerId: ID PT Bank Tabungan Negara (Persero)
    - contractId: Dapatkan dari kontrak aktif PT Bank Tabungan Negara (Persero) (jika perlu, buat kontrak versi baru untuk periode tersebut)
    - contractNumber: Sesuaikan dengan kontrak PT Bank Tabungan Negara (Persero) (biarkan kosong jika tidak ada)
    - invoiceNumber: Gunakan nilai yang sama atau sesuaikan dengan pola nomor invoice PT Bank Tabungan Negara (Persero)
    - periodStartDate, periodEndDate: Gunakan nilai yang sama
    - dueDate, paidAt: Gunakan nilai yang sama
    - amount: Gunakan nilai yang sama
    - packageType, ratio, coreTotal: Gunakan nilai yang sama
    - remarks: Tambahkan catatan bahwa ini adalah salinan dari arsip PT Karya Teknik Mulia (opsional)
    - scheduleStatus: "history"
  - Gunakan API `api.invoices.create()` untuk membuat arsip baru.
  - Jika ada file yang diunggah di arsip asli, unggah ulang file tersebut ke arsip baru menggunakan `uploadFileForRecord()`.

### 4. Verifikasi
- Setelah membuat semua arsip, kembali ke halaman detail PT Bank Tabungan Negara (Persero).
- Buka tab "Invoice" dan pastikan arsip yang baru dibuat muncul di bagian riwayat.
- Periksa bahwa jumlah arsip sesuai dengan yang diharapkan.

## Catatan Penting
- Pastikan PT Bank Tabungan Negara (Persero) memiliki kontrak yang aktif yang sesuai dengan periode arsip yang sedang disalin. Jika tidak, mungkin perlu membuat versi kontrak terlebih dahulu.
- Operasi ini hanya menyalin metadata arsip, bukan mengubah data transaksi aktual seperti pembayaran atau tagihan aktif.
- Selalu buat backup sebelum melakukan operasi massal data jika memungkinkan.
- Jika ada kesalahan selama proses, catat pesan error dan lanjutkan ke arsip berikutnya.

## Estimasi Waktu
- Pengambilan data: 5 menit
- Pembuatan arsip per record: 10-30 detik tergantung pada jumlah arsip dan ukuran file
- Verifikasi: 5 menit
